config

显示在commit记录中会出现的用户名

git config --global user.email

显示在commit记录中会出现的用户邮箱

git config --global user.email new_email

更改邮箱，new_email是新的邮箱。邮箱甚至可以不是真实邮箱

区别详见

clone

git clone https://github.com/username/repository_name

下载远端库上的项目到本地，会创建一个名为repository_name的目录

rm

git rm -r *

-r是递归指令，*是匹配任意文件名的正则表达式。这两个输入合起来的意思就是删除该目录下所有文件。
同时这个操作并不会删除不在git记录中的文件，如果删除一个不在git记录中的文件，命令会提示找不到该文件。
对于在git记录中的文件，即从git中删除，也会实际从本地中删除。

删除github上一个库下的所有文件

github网页端一次只能删除一个文件，但是有时候想删除一个库下的不少文件，包括文件夹，使用github的网页端就太麻烦了。可以使用Git命令，先将远程库clone到本地，然后修改本地的库，达到想要的效果后，上传到远端的库。
以现在的主页dddeadfish.github.io为例，具体操作为：

  1. 先从远端库上将这个项目下载下来，使用clone操作
    git clone https://github.com/dddeadfish/dddeadfish.github.io

    注意这这个地址，写成https://dddeadfish.github.io或者https://dddeadfish/dddeadfish.github.io这样的网址都是不对的(前者只是访问博客主页的地址)，如果不确定网址究竟是什么，可以到网页端的github库的主页，复制地址栏中的地址。

  2. 然后进入到该项目的目录里：
    cd dddeadfish.github.io

  3. 接下来使用删除操作将所有文件都删除掉：
    git rm -r *

  4. 要让这一系列删除生效，需要先提交修改：
    git commit -m "delete all files"

  5. 最后，将本地的库上传到远端库：
    git push origin master

    这个操作上传后，就可以在远端库的提交记录中看到这次操作提交的commit了，没有push前是没有的。这一步会有两次输入密码，第一次是弹出一个图形窗口，第二次是在git CMD中输入。第二次输入的时候，输入后密码是不显示的，可以使用ctrl+v来粘贴。
